## Описание

### Проблема

В настоящее время в России онлайн-банкинг является высокоразвитой средой. Пользователи могут выпустить банковскую карту без визита в отделение банка, взять кредит, выпустить e-sim, открыть вклад за считанные минуты. Крупные банки соревнуются друг с другом, переманивая своих клиентов большим кэшбэком, процентом на остаток по счету, длинным беспроцентным периодом. Как итог, пользователи выпускают множество карт разных банков, чтобы получать лучшие предложения по кэшбэку. Многие ИП используют несколько кредитных карт, чтобы покрывать временные расходы и увеличивать кредитный лимит.

Современный владелец бизнеса, или просто студент, который стремиться экономить и максимально эффективно получать бонусы с покупок, устанавливает 5-6 приложений мобильных банков. Очередной раз приходится открывать несколько из них по-очереди, чтобы проверить баланс, перевести деньги. FM синхронизирует информацию со всех банковских приложений, помогает планировать бюджет, выбирать оптимальную карту для оплаты следующей покупки

### Монетизация

- Встроенной рекламы в приложении быть не должно, так как это негативно скажется на удобстве пользователя и сервис потеряет смысл
- Оптимально, сделать сервис платным с пробным периодом. В бесплатной версии можно отключить ключевые функции (оплата в магазинах приложений, на сайте - классика)

Из чуть менее реалистичного, но возможного в будущем:

> Заручиться поддержкой крупной компании (оптимально Яндекс) для того, чтобы реализовать внутреннюю оплату, используя сервисы Я-Pay.  Добавить приложение в подписку Я-Плюс :-)

## План реализации

1. Собрать макет приложения для iOS, в котором будут реализованы простые функции и алгоритмы, стоять “заглушки” для оплаты и парсинга. Предложить алгоритмы “парсинга”, расшифровки СБП QR-code, перевода оплаты в банковские приложения

Таким образом, получится собрать заготовку для выполнения проекта, в котором будет четко обозначена работа всех функций

1. Собрать команду из
    1. Web-разработчика (Backend + Frontend) 
    2. UX-designer
    3. Аналитика данных 
    4. Android разработчика
    5. Специалиста по фин-теху
    
    (Некоторые из них могут не принимать полного участия в проекте, а лишь быть консультантами в нужной области)
    

Обсудить возможность оптимизаций и реализаций идей проекта, доработать макет и алгоритмы

1. Провести опрос среди целевой аудитории 
    
    Открыть доступ для использования макета, собрать обратную связь
    
2. Разработать структуру проекта (ООП таблица, интеграция сервисов)
    
    Сделать план в miro (полную структуру взаимодействия сервисов)
    
3. Найти ментора из сотрудников ВШЭ, который проконсультирует нас по поводу работы с компаниями, поиска сотрудников Яндекса, которые смогут помочь в интеграции сервисов оплаты
4. Реализовать (описанные в шаге “Фичи и реализации”) сложные фичи проекта (для backend) на базе арендованного сервера и web среды
    1. В это время frontend и ux-designer прорабатывают интерфейс для разных устройств
5. Интегрировать алгоритмы (с сервера) в мобильные приложения под разные платформы, вынести так же функции в Web-site
6. Подключить синхронизированную монетизацию под все платформы (на основе единого аккаунта)
    
    Разработать план завлечения пользователей путем скидок, бонусов
    
7. Запустить бесплатную альфа-версию и открыть доступ участникам первого опроса на фиксированный срок - на момент тестирования продукта 
8. Пройти цикл: [собрать отзывы - исправить проблемы] и подготовить финальный продукт
9. Запустить финальный продукт (дать тестировщикам бесплатный доступ или большую скидку, чтобы привлекать аудиторию) 
10. Искать поддержку крупных компаний, чтобы расширять проект
